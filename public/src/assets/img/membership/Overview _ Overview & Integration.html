Overview

Objectives: 

  • What is Angular?
  • Where does Angular fit in the http request and response cycle?
  • What are the benefits of using Angular?

Welcome to Angular! First, let's answer some of the most common and important questions.

What is Angular?

Angular is a JavaScript Front-End Framework that helps developers create Single Page Applications. A Single Page Application (SPA) is exactly what it sounds like: there is only one page - index.html - that builds the entire application. To generate different views for the user, we will not ask our server to render different files anymore. Instead, we’ll just ask our server for data via an AJAX request, then use the browser to modify the index.html file that is already there. This is what we call manipulating the DOM, and you may remember doing this before with your AJAX and jQuery assignments.

What are the benefits of Angular?

If you’ve been reading and you're still wondering why you should try using Angular, you probably need a refresher on AJAX! AJAX is what makes the web such a delight to use.

Imagine a world without AJAX. Imagine scrolling through your favorite social media site, such as Facebook. You’re scrolling, scrolling, scrolling… and then you see a post that you just cannot resist commenting upon. You submit your comment. What happens next? Since you don’t have AJAX, the page refreshes! You’re back at the top of the page! You have to scroll through everything again and your user experience has just been ruined!

Fortunately, we live in a world with AJAX. AJAX is why your page stays right where it is when you post your comment so you can see it magically appear on the page. You may continue your scrolling adventure without interruption. AJAX is a dream come true for the user experience, but writing the code has been tedious for the developer. Now, Angular gives us great tools that streamline how we write our code and handle AJAX requests, so the developer experience can be a dream come true, too.

Still need more benefits to be convinced to try Angular? Okay. Angular shifts the burden of processing data into html from our server to the client! Angular is a Front-End Framework, meaning it lives on our users’ browsers. We no longer need server-side rendering! We’ll be using Client Side Templating instead, which means each browser will process data into html. The server gives the browser index.html and a bundle of files to establish our Angular logic, but after that, the server only needs to provide data so that Angular can do its job. This also makes for a better user experience because it’s faster! This is especially helpful when the website is getting lots of traffic. One browser can process one page much faster than one server can process thousands of pages.

Since Angular is particularly useful for larger applications, it is designed to be used by large teams of developers. Therefore, Angular is structured and modularized, which means everyone on the team has to follow a pre-determined folder and file structure. Angular is an MVC framework, so the files and folders are divided into three major sections:

  • Models (used to handle data - referred to as Services in Angular)
  • Views (used for HTML/CSS - all pages that are visual)
  • Controllers (responsible for all the logic)

Angular is also component-based, which is a way to break the web page into different independent sections, a method that is shared with other Front-End Frameworks, such as React. Learning Angular will give you a solid foundation to learn other Front-End Frameworks.

To recap

There are a lot more benefits we could get into about Angular, but we're sure you're ready to just get started. You'll get a smoother user experience with a single-page app, you'll reduce the load on your server with client-side templating, and you'll have tools that streamline how you handle AJAX requests and work with a team!

** PUBLIC SERVICE ANNOUNCEMENT **

Be mindful that Angular and AngularJS are NOT THE SAME THING. While they are both currently being supported, Angular is a complete rewrite of AngularJS. Their syntax and approach are very different. This is going to help you level up your google-fu! While reading documentation and other resources, if you find solutions for a number version of Angular (2, 4, 5, etc.), you should be fine for the most part, but solutions for AngularJS will lead you down the wrong path. Also, while googling, include -angularjs (note the minus sign) in your query to exclude hits for AngularJS.

** END OF PUBLIC SERVICE ANNOUNCEMENT **
